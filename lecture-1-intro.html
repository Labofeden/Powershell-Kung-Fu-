Lecture 1: Introduction to PowerShell & Security

Pro Tip: This course teaches PowerShell like a martial art—every command must be precise, controlled, and secure.

Why PowerShell?

  • Automation: Eliminate repetitive tasks with scripts
  • Power: Control Windows, Linux, and macOS systems
  • Security: Built-in features like execution policies and logging
Warning: Great power comes with great responsibility. Misused PowerShell can be a hacker's weapon.

Your First Spell (Command)

Security Foundation: Execution Policies

PowerShell restricts script execution by default to prevent attacks:

# Check current policy
Get-ExecutionPolicy

# Set a safer policy (recommended)
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

Challenge: The Initiation Test

  1. Write a script that displays your username and execution policy
  2. Make your username appear in green and policy in yellow
# Solution:
$user = $env:USERNAME
$policy = Get-ExecutionPolicy
Write-Host "User:" -NoNewline
Write-Host " $user" -ForegroundColor Green
Write-Host "Policy:" -NoNewline
Write-Host " $policy" -ForegroundColor Yellow
